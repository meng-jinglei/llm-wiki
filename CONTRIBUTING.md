# 贡献指南

欢迎贡献！llm-wiki 是一个技能定义仓库，贡献可以围绕工作流、规则、模板和文档展开。

## 接受哪些贡献

- 工作流改进（新增、优化现有流程）
- 规则增强（编译检查项、安全约束）
- 模板优化（页面模板、初始化模板）
- 文档修正和翻译
- 示例工作区完善

## 贡献流程

1. Fork 本仓库
2. 创建功能分支：`git checkout -b feat/xxx` 或 `fix/xxx`
3. 提交变更：遵循 `type: description` 格式（feat / fix / docs / refactor / chore）
4. Push 并创建 PR
5. 等待 review

## 测试要求

在提交 PR 前，请在 Claude Code 中实际触发相关工作流进行验证：

- 用真实的 URL、文件或提问测试
- 确认输出符合预期格式
- 确认日志正确写入了 `log.md`

## 设计原则

- **轻薄应用层。** 不引入新存储格式、中间件或重型基础设施
- **Markdown 优先。** 所有持久化数据以 Markdown 存储
- **LLM 是编译器。** 工作流执行者是 LLM，不是外部运行时
- **语义约定 > 形式语法。** 优先用 Markdown 列表、wikilink 等惯例表达结构

## 报告 Issue

请在 Issue 中提供：

1. Claude Code 版本
2. 操作系统
3. 触发的工作流
4. 预期行为 vs 实际行为
5. 相关文件路径或日志片段
