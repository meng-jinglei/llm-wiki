# 抽象解释

静态程序分析的**统一数学框架**。由 Patrick Cousot 和 Radhia Cousot
于 1970 年代末形式化。

## 核心思想

将程序的**具体语义**（所有可能的执行轨迹和状态）映射到
**抽象语义**（简化的、可计算的版本），通过抽象域上的不动点计算
获得具体语义的安全近似。

```
具体域 C  ←──────────→  抽象域 A
           α (抽象化)
           γ (具体化)

Galois 连接: α 和 γ 互为伴随，保持信息偏序
```

## 关键概念

| 概念 | 说明 |
|------|------|
| **具体语义** | 描述程序实际执行的所有信息（不可计算） |
| **抽象语义** | 简化版本，如仅追踪整数符号而非精确值 |
| **Galois 连接** | α 和 γ 之间的伴随关系，保证抽象的可靠性 |
| **最小不动点** | 抽象域上的迭代固定点，近似具体不动点 |
| **Widening (∇)** | 加速收敛算子——当格无限高时牺牲精度保证终止 |
| **Narrowing (Δ)** | 在 widening 后提高精度 |

## 抽象域示例

| 抽象域 | 追踪信息 | 复杂度 |
|--------|---------|--------|
| 符号域 | {−, 0, +} | O(n) |
| 区间域 | [a, b] | O(n²) |
| 多面体域 | 线性约束 | O(2ⁿ) |
| 八边形域 | ±x ± y ≤ c | O(n³) |

## 实例：符号分析

```c
int x = 5;        // x: +
int y = -3;       // y: −
int z = x + y;    // z: + + − = ⊤（未知符号，因为结果可能为正或负）
```

抽象解释在此计算 `{+, −} → ⊤`——可靠地告诉我们符号结果不可确定。
