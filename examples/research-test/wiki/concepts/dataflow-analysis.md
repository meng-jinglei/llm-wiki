# 数据流分析

静态程序分析中最经典的技术。基于控制流图 (CFG) 和格论，
通过迭代求解数据流方程收集程序点的可能值信息。

## 核心机制

对于 CFG 中每个基本块 b：
```
in_b  = join({ out_p | p ∈ predecessors(b) })
out_b = transfer_b(in_b)
```

迭代直至所有 `in_b` 和 `out_b` 不再变化（到达不动点）。

## 收敛条件

- 值域必须是**有限高度的偏序**
- 传递函数和 join 操作必须是**单调**的
- 满足上述条件时，迭代必定终止于不动点（Kleene 不动点定理）

## 经典分析类型

| 分析 | 方向 | 格 | 用途 |
|------|------|-----|------|
| 可达定义 | 前向 | 幂集格 | 哪些赋值可能到达某点 |
| 活跃变量 | 后向 | 幂集格 | 哪些变量之后可能被使用 |
| 可用表达式 | 前向 | 幂集格 | 哪些表达式一定已计算 |
| 常量传播 | 前向 | 平坦整数格 | 变量是否恒为常数 |
| 区间分析 | 前向 | 区间格 | 变量可能的取值范围 |

## 与抽象解释的关系

数据流分析是抽象解释的一个具体实例：
- 数据流分析的格 = 抽象解释的抽象域
- 传递函数 = 抽象语义函数
- join = 最小上界 (⊔)
- 不动点 = 最小不动点近似
