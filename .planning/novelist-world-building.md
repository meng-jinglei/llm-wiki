# 世界观工程 — 研究课题

## 为什么世界观是独立课题

网文的世界观不是"背景设定"——它是**规则引擎**。角色在这个引擎里行动，剧情被引擎约束。
好的世界观（《神秘复苏》的鬼的规则、《诡秘之主》的序列体系）本身就是核心竞争力。

"世界观超一流，人物塑造不入流"——这句话说明：**在网文评价体系中，世界观可以独立于人物和情节成为主要卖点**。

## 核心命题

### 1. 世界观设计深度

一个好的网文世界观需要回答：

| 层面 | 问题 |
|------|------|
| **力量来源** | 这个世界的力量从哪里来？谁可以使用？ |
| **等级体系** | 力量如何分等级？每个等级有何质变？ |
| **代价与限制** | 使用力量有什么代价？有什么不可以做的事？ |
| **历史纵深** | 世界有怎样的历史？历史如何影响现在？ |
| **地理政治** | 势力如何分布？冲突如何产生？ |
| **社会结构** | 超凡者和平民如何共处？超凡如何影响社会？ |
| **隐藏真相** | 世界有什么不为人知的秘密？ |

### 2. 创新评定

如何判断一个世界观"够不够新"？

**新颖度光谱**：
```
完全模板化 ←──────────────────────→ 前所未见
   │              │              │
普遍玄幻世界观   诡秘之主序列   神秘复苏鬼的规则
(斗气+魔法)    (SCP+克苏鲁)  (游戏式规则)
```

评定方法：
- 核心规则是否可以追溯到更早的作品？（溯源）
- 组合方式是否产生了新意？（旧元素新组合也是创新）
- 是否有"只有这个世界才会发生"的情节？（规则驱动剧情）

### 3. 逻辑自洽验证

最常出问题的地方：

| 问题类型 | 示例 |
|---------|------|
| **能力膨胀** | 前期说"只有三个人能达到"，后期遍地走 |
| **规则遗忘** | 设定了代价但从未兑现 |
| **前后矛盾** | 第3章说X不行，第50章主角用X解决问题 |
| **例外过多** | 规则对主角总是不适用 |
| **数值崩坏** | 初期说1个斗圣能灭100个斗尊，后期主角1v100 |

**自动化检测方案**：
- `world-consistency` 工作流：扫描 wiki 中的规则声明 → 检测新章节是否违反
- 类比 llm-wiki 的 `compile` 工作流，检测的不是"来源锚点缺失"而是"规则违反"

### 4. 世界观 Wiki 化结构

```
wiki/novels/<slug>/world-building/
├── _index.md              ← 世界观总览 + 创新评定
├── power-system.md        ← 力量体系
│   ├── 等级定义
│   ├── 晋升条件
│   ├── 能力边界
│   └── 代价/副作用
├── geography.md           ← 地理/势力分布
├── history.md             ← 纪元/重大事件/隐藏真相
├── organizations.md       ← 组织/势力/关系网
├── magic-rules.md         ← 魔法/超自然规则（如有）
├── society.md             ← 社会结构/超凡与凡人的关系
└── consistency-check.md   ← 逻辑自洽验证记录
```

每个页面包含：
- **规则声明**：可被 compile 机械验证的原子化规则
- **实例引用**：小说中规则被展示/使用的具体场景
- **边界测试**：规则在极端情况下的表现

### 5. 世界观驱动的写作

agent 写作时，世界观 wiki 不是"参考"而是**约束**：

```
写作前: query world-building wiki → 确认规则边界
写作中: 每涉及超凡能力时 → 检查是否违反规则
写作后: world-consistency 检查 → 规则遵守度报告
```

这比"让 AI 自行发挥世界观"可靠得多——因为规则被显式定义了。

## 研究计划

### Phase 1：分析已有作品的世界观
- 斗破苍穹：传统玄幻模板（可作为"模板基准"）
- 诡秘之主：序列体系 + 非凡特性守恒定律
- 神秘复苏：鬼的规则（极简主义世界观）

### Phase 2：建立世界观评定框架
- 设计创新度评分标准
- 设计逻辑自洽检测算法
- 设计世界观 wiki 模板

### Phase 3：世界观生成能力
- 从分析过的世界观中提取"规则设计模式"
- agent 能否根据模式生成新世界观？
- agent 能否生成有"诡秘味道"但不同的序列体系？

### Phase 4：世界观驱动的长篇写作
- 100 章写作中，世界观能否保持一致？
- 能否在写作过程中"揭示"隐藏真相（像真正的连载一样）？

## 关键挑战

| 挑战 | 难度 | 说明 |
|------|------|------|
| 逻辑自洽自动检测 | 中 | 可以参照 compile 的声明冲突检测 |
| 创新能力评定 | 高 | 需要人工判断，LLM 只能做辅助 |
| 世界观"揭示"节奏 | 高 | 什么时候揭露什么——这是叙事艺术 |
| 规则驱动剧情 | 极高 | 让剧情从规则中自然生长，而非编剧强行推动 |

## 与 llm-wiki 的对应

| llm-wiki 概念 | 世界观工程对应 |
|-------------|-------------|
| `## 关键声明` | 规则声明（"序列9→序列8需要XX魔药"） |
| claims 三元组 | 规则三元组（序列, 晋升条件, 魔药配方） |
| compile 矛盾检测 | 规则违反检测 |
| source map | 世界观元素地图 |
| ingest | 从原文提取规则 |

世界观工程不需要新范式——它只是 llm-wiki 编译范式在"世界规则"这个领域的自然延伸。
