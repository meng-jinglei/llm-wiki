# 验收框架 — 三层验证 + 反AI专项

## 概述

文风文件的验收不是"看起来像不像"——必须有可量化、可复现的验证流程。
三层验证从自动化到人工，逐层递进。

## 第一层：定量指标对比（自动化，零成本）

### 方法
`style_metrics.py` 同时扫原作和 AI 生成物，对比核心指标。

### 对比指标

| 指标 | 测量内容 | 容差 |
|------|---------|------|
| 句长分布 | avg + sigma | ±15% |
| 感叹号密度 | /千字 | ±20% |
| 对话比例 | 对话行占比 | ±15% |
| 独句段比例 | 独句段/总段落 | ±20% |
| 章末模式 | 感叹号/省略号/悬念比例 | 模式一致即可 |
| 段落长度分布 | avg + sigma + quartiles | ±20% |

### 验收标准
6 个指标中至少 5 个在容差范围内 → 通过。否则 profile 需要调整。

## 第二层：LLM 盲测（中等成本，定期跑）

### 测试 A：区分测试
```
取 5 段原作摘录 + 5 段 AI 生成物 → 随机排列
→ 给评判 LLM (非写作 LLM): "哪些是<作者名>原作？"
→ 评判 LLM 无法可靠区分 → 通过
→ 评判 LLM 可准确区分 → 未通过，记录区分依据供 profile 调整
```

**验收标准**：盲判准确率 ≤ 60%（随机 = 50%）。

### 测试 B：维度评分
```
评判 LLM 对 AI 生成物在八个维度上评分 (1-5):
  句法肌理:  ?/5
  词汇光谱:  ?/5
  叙事节奏:  ?/5
  氛围质地:  ?/5
  场景生态:  ?/5
  套路指纹:  ?/5
  对话纹理:  ?/5
  世界观一致性: ?/5
```

**验收标准**：均分 ≥ 3.5，无维度低于 2。

## 第三层：人类验收（最终关卡）

### 测试 A：图灵测试
原作与 AI 生成章节混排，人类读者判断哪些是 AI 写的。
**标准**：人类判准率 ≤ 随机水平。

### 测试 B：裸读测试
给人类读者看 AI 生成章节（不说来源），观察是否会自发指出"像 AI 写的"。
**标准**：不被主动识破。

### 测试 C：写作实战
给目标用户一个 profile 文件，用 profile 指导写作一章。
观察：能不能用？写出来像不像？有没有帮助？
**标准**："能用"即通过。

## 反 AI 检测专项验收

基于 `wiki/novels/<slug>/ai-gaps.md` 中提取的反 AI 信号，逐条检查：

| 信号 | 检查方式 | 标准 |
|------|---------|------|
| 感叹号非理性使用 | AI 物中陈述句+！的比例 | ≥ 原作的 70% |
| 围观群众"弹幕" | 碎片化、不推进剧情的对白片段 | 每章 ≥ 1 处 |
| 一字句/短动作句 | "呸。""草！"类单字段出现频率 | ≥ 原作的 60% |
| 句子"粗粝感" | 语法的微小不完美 | 每章 ≥ 2 处 |
| 重复用词 | 标志性词汇的重复率 | ≥ 原作的 80% |
| 第四面墙 | 作者直接跟读者说话 | 符合时代（现代网文可选）|

**标准**：6 个信号中 ≥ 4 个可检测。

## 令牌：集成到管线

```
style-analyze → style-profile → 生成 profile
                                    ↓
                            agent 用 profile 写一章
                                    ↓
                    ┌── 定量对比 (自动, layer 1)
                    ├── LLM 盲测 (半自动, layer 2)
                    └── 人类验收 (手动, layer 3)
                                    ↓
                            不通过 → 反馈原因 → 调整 profile → 重试
                            通过   → profile 标记 verified
```
