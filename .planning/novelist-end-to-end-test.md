# 端到端实战测试 — 诡秘之主

## 目标

对《诡秘之主》（爱潜水的乌贼）进行完整的文风研究 → 生成 profile → agent 用 profile 写 20 章 → 三层验收。

## 测试范围

| 阶段 | 内容 | 预计工作量 |
|------|------|-----------|
| 1. 准备 | TXT 下载 + 全量指标扫描 | 已完成下载, 待扫描 |
| 2. 文风分析 | 双层混合分析（55章战略 ingest） | ~8-10 sessions |
| 3. 世界观分析 | 序列体系 + 历史 + 地理 + 组织 wiki | ~5-8 sessions |
| 4. Profile 生成 | 从 wiki 编译文风+世界观 profile | ~2 sessions |
| 5. 写作实战 | agent 用 profile 写 20 章 | ~5-10 sessions |
| 6. 验收 | 三层验证 | ~2 sessions |

## 阶段详解

### 阶段 1：准备
- [x] 诡秘之主 TXT 已下载 (9MB, 446万字)
- [ ] `style_metrics.py` 全量扫描 → 定量指纹 + 词汇聚类 + source map
- [ ] 与斗破苍穹定量对比（建立基线）

### 阶段 2：文风分析
按双层混合方法论执行：
- 计算层：style_metrics.py 产出自适应词汇聚类 + 极端章定位
- Wiki 层：战略 ingest ~55 章（按 source map 优先级）
- 产出：8 维度分析 + 角色/情节/场景 wiki

**诡秘的特殊挑战**：
- 乌贼的文风比土豆复杂得多——理性主角、多身份切换、SCP式序列体系
- 需要特别注意：氛围质地（克苏鲁式冷峻）和叙事节奏（波浪型）
- 对话功能分布预计与土豆完全不同（信息传递 >> 情绪释放）

### 阶段 3：世界观分析
诡秘之主的世界观是其核心竞争力。需要专项分析：

**序列体系**：
- 22条神之途径 × 序列9→序列0
- 魔药配方 + 晋升仪式 + 失控风险
- 序列名称的命名美学（"占卜家""无面人""诡秘侍者"）

**历史与地理**：
- 五个纪元的历史
- 北大陆（鲁恩/因蒂斯/弗萨克）+ 南大陆
- 灵界、星界、冥界

**组织与势力**：
- 正神教会（七神 + 黑夜/风暴/蒸汽等）
- 隐秘组织（塔罗会/心理炼金会/玫瑰学派等）
- 王室与军方

**逻辑自洽验证**：
- 序列能力是否有矛盾使用？
- 魔药副作用是否被遵守？
- 非凡特性守恒定律是否成立？

### 阶段 4：Profile 生成
从 wiki 编译两个输出文件：
1. `profiles/诡秘之主_文风.yml` — 句法、节奏、氛围、对话等
2. `profiles/诡秘之主_世界观.yml` — 序列规则、地理约束、组织关系

### 阶段 5：写作实战
agent 用两个 profile 写 20 章诡秘之主风格的同人/续写。
每章写作流程：
```
读 profile → 读世界观 wiki → 读上一章摘要 → 写 → 更新 wiki
```

### 阶段 6：验收
三层验证执行——定量对比 → LLM 盲测 → 人类（你）判断。

## 关键风险

| 风险 | 缓解 |
|------|------|
| 世界观过于复杂，wiki 化耗时过长 | 优先核心序列体系，其他按需扩展 |
| 乌贼文风复杂，profile 不够精准 | 迭代调整，每 5 章验一次 |
| 20 章 AI 写作质量不稳定 | 逐章 review + compile 检查 |
| 人类验收主观性强 | 以你的判断为最终标准 |

## 时间线

```
当前 ──→ 诡秘之主全量扫描    (本 session)
      → 诡秘之主 Source Map
      → 诡秘之主 Wiki Ingest   (未来 sessions)
      → Profile 生成
      → 20 章写作
      → 验收
```
